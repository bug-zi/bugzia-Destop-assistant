## 多窗口架构重构（三窗口，已落地）

为解决"常驻长条使用时被拉成大矩形、破坏桌面装饰感"的问题（设计见 [`双窗口长条与结果浮层优化方案.md`](双窗口长条与结果浮层优化方案.md)），应用从单窗口迁移到**三窗口架构**。三个窗口都加载同一个 `index.html`，由 `App.tsx` 按 `getCurrentWindow().label` 路由到不同根组件。

| 窗口 | label | 声明方式 | 根组件 | 现状 |
|---|---|---|---|---|
| 主长条 | `main` | `tauri.conf.json` 静态声明（420×64，`maxHeight:72` 锁死长条） | `CommandCard` | **已固化为长条**：不再用 `expanded` 拉高、`result-panel` 已移除；仅渲染 `InputBar`（拖动手柄 + spark 切换结果框 + 输入框 + 设置齿轮）与一行短状态；AI 流式改由结果窗口承载 |
| 设置弹窗 | `settings` | 前端 `WebviewWindow` 动态创建（`features/settings/settingsWindow.ts`，460×640） | `SettingsWindow` → `SettingsPanel` | 已落地 |
| 结果浮层 | `result` | 前端 `WebviewWindow` 动态创建（`features/result/resultWindow.ts`，主窗口上下文定位） | `ResultWindow`（玻璃卡片 + 拖动 / 固定 / 关闭 + 复用 `ChatView` / `FileResultsView`） | 已落地：create-once-then-hide、近主窗口定位（上下边界修正）、`result:ready` / `replay` 握手、流式镜像、Esc / 固定 / 停止 / 清空、打开设置前自动隐藏未固定结果、**结果窗口几何（`result_x` / `result_y` / `result_w` / `result_h`）全部持久化**（Rust `WindowSettings` serde default + 手动 `Default`，向后兼容旧 `settings.json`；主窗口经结果窗口句柄 `onResized` / `onMoved` 写回 + 启动恢复） |

**窗口创建机制**：设置窗口由 `openSettingsWindow()` 用 Tauri JS 的 `WebviewWindow` 构造器按需创建（已存在则 show + focus），`main` 的 capability 含 `core:webview:allow-create-webview-window`。结果窗口同理由 `features/result/resultWindow.ts` 的 `ensureResultWindow` / `showResultWindow` / `hideResultWindow` / `positionResultWindowNearMain` 管控（create-once-then-hide，状态持久；定位与改尺寸跑在 main，因 ACL 按调用者窗口校验，仅 main 有 set-position / set-size）。三份 capability 按 label 分窗口（`capabilities/default.json` 给 main、`result.json`、`settings.json`）。

**设置同步机制**：设置窗口**不写 `settings.json`**，只编辑并广播 `settings:updated` 事件（debounce 400ms）；**主窗口是唯一写盘者**，收到事件后把 patch 与自己权威的 window bounds 合并再持久化——避免设置窗口尺寸被误存成卡片尺寸。Esc 关设置窗前会 flush 最后一笔。

**spark 切换结果框**：主长条左侧原拖动手柄拆为两元素——窄 `.grip`（点点 `GripIcon`，`data-tauri-drag-region`）负责拖动 + `.spark` 按钮（`SparkIcon`，点击切换结果窗，结果窗打开时填充高亮）。`CommandCard` 持 `resultOpen` 状态并经 `revealResult` / `concealResult` 收敛所有 show / hide 调用点（AI 回合 / `/file` / Esc / `command:close-result` / 开设置 / spark 切换），spark 始终反映结果窗真实可见性。

## 环境与开发须知

- **Rust 不在 PATH**：任何 `cargo` / `tauri` 命令前先 `export PATH="$HOME/.cargo/bin:$PATH"`。
- **开发**：`pnpm tauri dev`（后台常驻长进程）。
- **开发端口 1520**：`vite.config.ts` 与 `tauri.conf.json` 的 `devUrl` 均为 `1520`（`strictPort`，占用即报错退出）。**不是** 1420——1420 落在 Windows 动态保留端口段（1414–1513），绑定会抛 `EACCES`，故改用 1520。若 1520 被残留孤儿 dev 进程占用，按 PID 精确结束即可。
- **缓存差异**：`cargo check` 与 `pnpm tauri dev`（`--no-default-features`）**缓存不通用**——dev 首次可能全量重编约 1 分钟；但 plain `cargo run` 与 `cargo check` 共享缓存。

## 已确认的踩坑

1. **不置顶**：用户要求卡片**不置顶**（已删 `alwaysOnTop`），应被其它窗口覆盖。
2. **拖动区域**：`data-tauri-drag-region` 不能放在输入框/按钮祖先上，必须用与 input 同级的 `.grip` 手柄。
3. **keyring v3**：用 `delete_credential`（非 `delete_password`）；`NoEntry` 当作"无 Key"返回 None 而非报错；必须显式 `features=["windows-native"]`，否则回落内存 mock store、重启即丢 Key。
4. **窗口单位**：事件 payload 是物理像素，存盘前 `/scaleFactor` 转逻辑像素，与 `LogicalSize` 同空间，避免重启闪烁。
5. **serde rename 陷阱**：`#[serde(rename_all="camelCase")]` 放在 enum 上只重命名变体名，不下钻字段；`Done{full_text}` 需显式 `#[serde(rename="fullText")]`，否则前端 `e.data.fullText` 得 `undefined`、done 事件把已流式内容擦空。
6. **Channel 事件枚举必须 `pub`**：`ChatEvent` 出现在 `pub async fn chat(on_event: Channel<ChatEvent>)` 签名里，enum 不 pub 时 `generate_handler!` 会报 `type ChatEvent is private`（hard error）。
7. **别用 `| tail` 接 cargo 退出码**：`cargo check 2>&1 | tail` 的 `$?` 是 `tail` 的，会吞 cargo 失败。验证编译用 `cargo check 2>&1; echo "===EXIT:$?==="` 或 `${PIPESTATUS[0]}`。
8. **BaseURL 必须带 `/v1`**：填 `https://token.aiedulab.cn`（无 `/v1`）→ 请求 `/chat/completions` 命中网关前端 SPA 返 `text/html`，SSE 解析 0 条报错。OpenAI 兼容路径标准是 `/v1/chat/completions`，one-api/new-api/LiteLLM 同理。
9. **模型自我认知不可信**：以网关 API 回显的 `model` 字段为准，别问模型自己（曾因 gpt-5.5 自称 "GPT-5.1" 误判）。`ai.rs` 的 `Done` 已回传服务端真实 `model`。

**后端（`src-tauri/src/`）**
- `main.rs` — Tauri 入口
- `lib.rs` — Tauri Builder、命令注册（settings×5、chat / stop_chat / clear_context / get_messages / test_ai_connection、search_files / open_file / reveal_file）、托盘 `build_tray`、全局快捷键 `Alt+Space`、开机自启 `sync_autostart`、`.manage(ChatState)`
- `ai.rs` — OpenAI-compatible SSE 流式 + Channel 推送 + 上下文 / 停止 / 清空
- `file_search.rs` — 实时文件名搜索（`spawn_blocking` 遍历）+ `open_file` / `reveal_file`（#7）
- `settings.rs` — 配置读写（原子写）+ keyring API Key 安全存储

**窗口与权限**
- `tauri.conf.json` 仅静态声明 `main`（420×64，`maxHeight:72`）；`settings` / `result` 窗口由前端动态创建；开发端口 `1520`
- `capabilities/`：`default.json`(main)、`result.json`、`settings.json` 三份，按 label 分窗口授权
