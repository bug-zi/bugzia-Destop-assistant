# 桌面 AI 对话框工具制作方案

## 1. 产品定位

这个工具不是 Raycast、Kimi Work 或完整桌面 AI 客户端的替代品，而是一个常驻桌面的半透明 AI 搜索对话组件。

它的核心形态是：

> 一个可由用户自由摆放在桌面任意位置的半透明白色 AI Command Card，位置可锁定，支持接入自己的 OpenAI-compatible Base URL，与 AI 随时对话，同时支持本地文件搜索和浏览器搜索。

关键词：

- 常驻桌面
- 半透明白色玻璃卡片
- 自由摆放位置并可锁定
- 可 DIY 样式（颜色 RGB + 透明度可调）
- 可接入自定义 AI API
- 可对话
- 可搜索本地文件
- 可调用浏览器搜索
- 轻量、聚焦、不做复杂工作台

## 2. 目标用户与使用场景

目标用户是希望把 AI 入口固定在桌面上的 Windows 用户。

典型场景：

- 桌面上长期显示一个美观的 AI 输入框，兼具装饰和实用功能。
- 需要随时向自己的大模型 API 提问。
- 需要快速搜索本地文件。
- 需要快速把关键词交给浏览器搜索。
- 不希望使用 Raycast 那种功能很多、需要快捷键唤醒、样式不可深度调整的工具。

## 3. 核心需求

第一版只做两个核心功能。

### 3.1 AI 对话

支持用户配置自己的 AI 接口：

- OpenAI-compatible Base URL
- API Key
- Model
- System Prompt
- Temperature

支持的对话能力：

- 流式输出
- 连续对话
- 清空上下文
- 停止生成
- 多模型配置
- 默认中文回复
- API Key 本地安全保存

说明：

OpenAI-compatible Base URL 指的是 API 接口地址，例如：

```text
https://api.openai.com/v1
https://openrouter.ai/api/v1
http://localhost:1234/v1
https://你的中转平台域名/v1
```

不是管理后台地址，例如：

```text
https://token.aiedulab.cn/dashboard
```

程序最终请求的接口通常是：

```text
POST {Base URL}/chat/completions
```

### 3.2 搜索

搜索分为两类。

#### 本地文件搜索

第一版只做文件名搜索，不做全文内容索引。

搜索范围：

- Desktop
- Documents
- Downloads
- 用户自定义目录

搜索结果展示：

- 文件名
- 文件路径
- 文件类型
- 修改时间

支持操作：

- 点击打开文件
- 打开所在文件夹

#### 浏览器搜索

支持把输入内容交给默认浏览器搜索。

内置搜索引擎：

- Google
- Bing
- 百度
- Perplexity
- 自定义搜索引擎

## 4. 推荐技术栈

推荐采用：

```text
Tauri v2 + React + TypeScript + Rust
```

职责分工：

- React：负责界面、交互、主题、动效。
- TypeScript：负责前端状态、请求事件、组件逻辑。
- Rust：负责本地文件搜索、系统窗口控制、打开文件和 URL、配置读写、AI 请求代理。
- Tauri v2：负责桌面窗口、托盘、开机启动、全局快捷键、系统能力桥接。

选择 Tauri 的原因：

- 体积比 Electron 小。
- 适合做透明、无边框、置顶的桌面组件。
- 前端样式自由，适合做半透明桌面美化组件。
- Rust 后端适合做本地文件搜索和系统集成。
- 支持全局快捷键、开机自启、托盘、打开文件和 URL、持久化设置。

## 5. 桌面视觉风格

整体风格：

- 浅白色半透明玻璃
- 轻微毛玻璃模糊
- 颜色由用户自定义（RGB + 透明度）
- 深色文字
- 小圆角
- 低阴影
- 与桌面壁纸融合，不抢画面

默认主题采用浅白色：

```text
背景：rgba(255, 255, 255, 0.34)
边框：rgba(255, 255, 255, 0.40)
输入框：rgba(255, 255, 255, 0.72)
文字：#1f2428
弱文字：rgba(31, 36, 40, 0.58)
圆角：10px - 14px
模糊：blur(18px)
```

颜色自定义说明：

- 用户可在设置中调节背景 RGB 三个通道（R / G / B）。
- 用户可独立调节整体透明度（Alpha）。
- 不再内置固定主题色，默认浅白色，其余由用户自行调配。
- 文字颜色建议根据背景亮度自动切换深 / 浅，避免低对比度看不清。

## 6. 界面形态

建议提供两个布局模式。两种模式只是卡片形状不同，最终落在桌面的具体位置都由用户自由拖动决定，并可锁定。

| 模式 | 默认起始位置 | 默认尺寸 | 适合场景 |
| --- | --- | --- | --- |
| 时间旁卡片模式 | 桌面时间右侧 | 420 x 64，展开 420 x 420 | 更像桌面组件，与当前桌面风格统一 |
| 顶部搜索栏模式 | 屏幕上方居中 | 560 x 56，展开 560 x 360 | 更像 Raycast，但常驻桌面 |

推荐第一版默认使用：

> 时间旁卡片模式

原因：

- 与当前桌面已有卡片组件更协调。
- 不破坏桌面中心区域的壁纸视觉。
- 既像装饰组件，又能作为 AI 入口。
- 和右侧文件卡片形成同一套桌面 UI 语言。

## 7. 主界面结构

默认收起态：

```text
┌──────────────────────────────────────┐
│  ✦ Ask or search...              ⚙   │
└──────────────────────────────────────┘
```

展开态：

```text
┌──────────────────────────────────────┐
│  ✦ Ask or search...              ⚙   │
├──────────────────────────────────────┤
│  AI 回复 / 搜索结果 / 文件结果         │
│                                      │
└──────────────────────────────────────┘
```

交互原则：

- 没有输入时，只显示半透明输入栏。
- 输入后自动展开结果区域。
- 失焦后可以自动收起。
- 用户可以固定展开。
- 用户可以自由拖动位置到桌面任意处。
- 用户可以锁定位置，锁定后卡片不再响应拖动。
- 用户可以调整宽度和展开高度。

## 8. 输入与命令规则

默认输入直接发送给 AI。

示例：

```text
帮我解释一下 OpenAI-compatible Base URL
```

特殊前缀：

```text
? 樱花壁纸 4K
/web Tauri 透明窗口
/file 论文
/ai 帮我总结这段话
```

建议规则：

- 普通文本：AI 对话
- `?`：浏览器搜索
- `/web`：浏览器搜索
- `/file`：本地文件搜索
- `/ai`：强制 AI 对话

快捷键：

```text
Enter：发送给 AI
Ctrl + Enter：浏览器搜索
Alt + Enter：本地文件搜索
Esc：收起结果区
Ctrl + L：聚焦输入框
```

全局快捷键：

```text
Alt + Space 或 Ctrl + Space：聚焦桌面输入框
```

说明：

虽然工具是常驻桌面的，但仍建议保留全局快捷键，用于快速聚焦输入框。

## 9. 窗口行为

第一版使用普通 Tauri 透明无边框窗口。

建议窗口配置：

```text
transparent: true
decorations: false
resizable: true
skipTaskbar: true
alwaysOnTop: 可选
```

窗口能力：

- 可拖动
- 可锁定位置（锁定后忽略拖动）
- 可缩放
- 可置顶
- 可隐藏到托盘
- 可开机启动
- 可记住上次位置和大小
- 可设置透明度
- 可设置自动收起

第二版再考虑真正桌面嵌入模式。

真正桌面嵌入模式指的是将窗口挂到 Windows 桌面层级中，类似 Rainmeter 组件。这需要 Rust 调用 Windows API，例如处理 `Progman` 和 `WorkerW` 窗口层级。

这个能力建议后置，因为它涉及：

- Explorer 重启兼容
- 多显示器兼容
- 桌面图标层级
- 窗口焦点问题
- DPI 缩放问题

## 10. 设置项

第一版设置项保持克制。

### 外观设置

- 位置：自由拖动定位，记住上次位置
- 锁定位置：开关，开启后卡片不可拖动
- 背景颜色 RGB：R / G / B 三通道分别可调
- 透明度（Alpha）
- 宽度
- 展开高度
- 背景模糊强度
- 圆角
- 字体大小
- 是否置顶
- 是否自动收起

### AI 设置

- Provider Name
- Base URL
- API Key
- Model
- System Prompt
- Temperature
- 是否流式输出
- 默认是否保留上下文

### 搜索设置

- 默认搜索引擎
- 自定义搜索 URL
- 本地索引目录
- 忽略目录
- 最大搜索结果数量

### 系统设置

- 开机自启
- 托盘图标
- 全局快捷键
- 是否显示任务栏图标

## 12. 后端模块设计

### ai.rs

负责：

- 接收前端发送的 AI 请求
- 拼接 OpenAI-compatible API 请求
- 保存和读取模型配置
- 处理流式响应
- 将 token 增量推送给前端

### file_search.rs

负责：

- 扫描指定目录
- 建立文件名索引
- 查询文件
- 打开文件
- 打开所在文件夹

第一版可以先不做复杂索引，只做实时文件名搜索或轻量缓存。

后续再考虑 SQLite FTS。

### settings.rs

负责：

- 保存外观设置
- 保存 AI 设置
- 保存搜索设置
- 保存窗口位置和大小

API Key 应尽量使用安全存储，不建议明文保存在普通 JSON 中。

### window.rs

负责：

- 设置窗口透明
- 设置置顶
- 设置窗口位置
- 锁定 / 解锁窗口位置（锁定后忽略拖动）
- 保存窗口大小
- 控制收起和展开
- 处理托盘菜单
- 处理全局快捷键

## 13. 第一版开发顺序

建议按以下顺序开发：

1. 初始化 Tauri v2 + React + TypeScript 项目。
2. 做透明无边框窗口。
3. 做半透明 Command Card UI。
4. 实现拖动、缩放、记住位置。
5. 实现设置页基础框架。
6. 接入 OpenAI-compatible API。
7. 实现流式 AI 输出。
8. 实现清空上下文和停止生成。
9. 实现浏览器搜索。
10. 实现本地文件名搜索。
11. 实现托盘菜单。
12. 实现全局快捷键聚焦。
13. 实现开机自启。
14. 实现颜色（RGB + 透明度）自定义、位置自由摆放与锁定。

## 14. 第一版不做的内容

为了避免变成另一个复杂 Raycast，第一版明确不做：

- 插件市场
- MCP
- 知识库
- 网页总结
- 复杂 Agent
- 文件全文索引
- 多窗口工作区
- 团队协作
- 云同步
- 复杂自动化工作流

这些功能可以后续扩展，但不应进入第一版。

## 15. MVP 验收标准

第一版完成后，应满足：

- 桌面上能长期显示一个半透明 AI 输入框。
- 界面为浅白色半透明玻璃，能与桌面壁纸融合。
- 可以配置 OpenAI-compatible Base URL、API Key 和模型名。
- 可以正常与 AI 流式对话。
- 可以清空上下文。
- 可以通过 `?` 或 `/web` 调用浏览器搜索。
- 可以通过 `/file` 搜索本地文件。
- 可以点击打开文件。
- 可以把卡片自由摆到桌面任意位置，并可锁定位置。
- 可以调节背景颜色（RGB）和透明度。
- 可以调整大小。
- 可以开机自启。
- 可以通过全局快捷键聚焦。

## 16. 最终建议

第一版就做：

> 浅白色半透明 AI Command Card，位置由用户自由摆放并可锁定。

默认形态是一个常驻桌面的输入框。普通输入走 AI，对 `?` 和 `/file` 做搜索。整体视觉为浅白色半透明玻璃，强调轻、美、常驻、位置自由、颜色可 DIY、可接自己的 API。

这个版本一旦跑起来，就已经和 Raycast、Chatbox、Cherry Studio 形成明显区别。
