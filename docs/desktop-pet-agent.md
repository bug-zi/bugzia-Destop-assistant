# 桌宠智能体开发文档

## 当前定位

桌宠是常驻桌面的克鲁鲁·采佩西交互式桌宠形态。她以《终结的炽天使》中的吸血鬼女王身份暂居在 Bugzia 桌面上，但不会复刻原作台词或编造未确认的原作剧情。目标是形成“动作 + 语料 + AI 即兴 + 记忆”的桌面陪伴智能体。

## 当前开发阶段

- 已完成本地动作语料：启动、闲置、点击、双击、拖拽、放下、入睡、唤醒、聊天、投喂。
- 已完成结构化 AI 即兴：复用项目现有 `ask_once`，返回台词、动作和情绪，不污染主聊天上下文。
- 已完成短期记忆：运行期记录最近互动和互动次数，重启清空。
- 已完成显式长期偏好：只在用户明确要求时保存到 `localStorage`。
- 已完成设置化：AI 即兴开关、AI 触发间隔、双击对话输入开关。
- 已完成搜索框监听点评：桌宠接收主搜索框输入预览，并用本地规则吐槽。
- 已完成情绪视觉反馈：台词气泡、角色投影、表情标记和轻量姿态动效会随当前情绪变化。
- 已完成提示优先级：闲置、输入点评、系统状态、用户交互不会互相随意抢气泡。
- 已完成 AI 不可用提示：AI 调用失败时会显示轻量 fallback 台词。
- 已完成聊天体验优化：提交后显示思考状态，AI 回复有超时保护，空内容不触发互动。
- 已完成拖拽稳定化：拖拽动画保持播放，窗口移动使用系统光标物理坐标轮询，避免事件丢失和起步偏移。
- 已完成桌宠窗口权限补齐：`pet` capability 允许前端调用 `setPosition` 移动桌宠窗口。
- 当前精灵图运行时接入的是 `assets/pet/vampire-sprite-v1/runtime/` 下的等宽帧条；`sheets/` 下还有更多草稿动作，暂未直接接入运行时。

## 使用说明

### 基础交互

- 单击桌宠：触发 `happy` 动作，并显示点击台词。
- 双击桌宠：触发 `surprise` 动作，并在启用“双击对话输入”时打开输入框。
- 输入框回车：把文本发送给桌宠，桌宠用人设提示词生成短回复。
- 输入框 Esc 或点击 `×`：关闭输入框。
- 拖拽桌宠：触发 `drag` 动作。
- 拖拽过程中：`drag` 动作保持循环播放，窗口位置由系统光标轮询更新。
- 松开拖拽：触发 `drop` 动作。
- 长时间无交互：触发 `sleep_start`，随后进入 `sleep`。
- 睡眠中点击：触发 `wake`。

### 文件投喂

- 把文件拖到桌宠窗口上：桌宠显示投喂提示。
- 松开文件：调用后端 `pet_eat_files`，将文件送入系统回收站。
- 全部成功：显示投喂成功台词。
- 全部失败：显示投喂失败台词。
- 部分失败：显示成功台词并提示失败数量。

### 搜索框监听点评

- 主窗口搜索框输入变化时，会广播 `pet:input-preview` 事件给桌宠。
- 事件包含输入文本、解析后的命令模式、时间戳。
- 桌宠只使用本地规则点评，不会把搜索框内容发送给 AI。
- 少于 4 个字符不反应。
- 3 秒内最多点评一次。
- 桌宠睡眠、拖拽、锁定时不反应。

当前规则：

- `/file`：触发 `mocking`，吐槽文件搜索。
- `/weather`：触发 `mocking`，吐槽天气查询。
- `/trans`：触发 `annoyed`，吐槽翻译。
- `/web`：触发 `curious`，催促用户去查。
- `/note`：触发 `happy`，认可便签行为。
- `/help`：触发 `mocking`，吐槽查命令。
- 输入包含问号：触发 `curious` 点评。
- 输入包含“累、困、烦、不想、摆烂、崩溃、难受”：触发 `protective` 关心式提醒。
- 输入包含“谢谢、太好了、漂亮、成功、搞定、完成”：触发 `happy` 开心反馈。
- 普通 AI 输入超过 40 字：触发 `curious` 长问题吐槽。

### 桌宠聊天

- 双击桌宠打开聊天输入框。
- 输入内容后按 Enter：关闭输入框，显示“稍等，本女王正在想。”，随后等待 AI 回复。
- AI 回复成功：显示 AI 生成的台词，并按返回的动作和情绪更新桌宠。
- AI 回复超过 20 秒或调用失败：显示 AI 不可用提示。
- 输入空内容后按 Enter：只关闭输入框，不触发动作、记忆或台词。
- 按 Esc 或点击 `×`：关闭输入框并清空输入。
- AI 回复回来前如果发生新的桌宠提示，旧回复不会覆盖当前提示。

### 长期偏好记忆

长期偏好只在用户明确说出偏好时保存，不会自动保存普通聊天内容。

当前支持的表达：

- `以后叫我小张`
- `称呼我主人`
- `我的名字是小张`
- `回复短一点`
- `说短一点`
- `回复正常`
- `清除记忆`
- `忘记偏好`

保存位置：

- 浏览器本地存储 `localStorage`
- key: `bugzia.pet.preferences.v1`

## 动作与触发

| 动作 | 触发 |
| --- | --- |
| `wave` | 桌宠加载完成 |
| `idle` | 默认状态 |
| `blink` | 闲置眨眼定时器 |
| `happy` | 单击、投喂成功、聊天提交 |
| `surprise` | 双击 |
| `drag` | 拖拽桌宠 |
| `drop` | 松开拖拽 |
| `sleep_start` | 超过自动入睡时间 |
| `sleep` | 入睡动作完成后保持 |
| `wake` | 睡眠中被交互唤醒 |

AI 和搜索框点评可以返回更细的动作意图。当前运行时优先播放对应的精灵动作帧；`mood` 只负责气泡、投影和轻量表情层，避免 CSS 身体动画和精灵帧互相抢动作。

| 动作意图 | 当前精灵动作 | 用途 |
| --- | --- | --- |
| `idle` | 不主动切动作 | 普通闲聊、普通回复 |
| `happy` | `happy` | 夸奖、感谢、投喂、积极反馈 |
| `surprise` | `double_surprise` | 惊讶、双击、强提示 |
| `wake` | `wake` | 睡眠中唤醒 |
| `annoyed` | `annoyed` | 不耐烦、被冒犯、拖拽抱怨 |
| `curious` | `curious` | 疑问、搜索、探索 |
| `protective` | `protective` | 护短、安慰、命令式关心 |
| `mocking` | `mocking` | 嘲讽、得意、轻微吐槽 |

## 情绪视觉反馈

情绪反馈不再承担身体动画。只要存在合适的动作意图，就交给精灵帧播放；情绪层只保留气泡颜色、投影和表情标记。

| 情绪 | 来源 | 视觉表现 |
| --- | --- | --- |
| `neutral` | 闲置、本地默认 | 白色气泡，普通阴影 |
| `pleased` | 启动、单击、聊天、投喂成功 | 暖粉气泡，柔和粉色投影，`♪` 表情标记 |
| `annoyed` | 双击、拖拽、翻译吐槽、睡醒 | 淡红气泡，偏强红色投影，`!` 表情标记 |
| `curious` | 投喂悬停、搜索框问题点评、长问题 | 淡紫气泡，轻微紫色投影，`?` 表情标记 |
| `sleepy` | 入睡提示 | 低对比气泡，`Z` 表情标记，角色透明度降低 |
| `protective` | 疲惫关键词、AI 判断为护短或关心 | 深玫红气泡，白色文字，`!!` 表情标记 |
| `mocking` | 放下、投喂失败、文件/天气/help 吐槽 | 冷灰紫气泡，锐一点的投影，`...` 表情标记 |

表情标记由 `MOOD_EXPRESSION` 控制，只作为当前 mood 的可视化增强，不参与事件优先级，也不拦截鼠标事件。

## 提示优先级

桌宠气泡统一经过 `showPetNotice` 显示，避免多个事件同时覆盖。每条提示默认显示 2.5 秒。

| 优先级 | 来源 | 行为 |
| --- | --- | --- |
| `ambient` | 闲置随机说话 | 不覆盖搜索框点评、系统状态、用户交互 |
| `input` | 搜索框输入点评、投喂悬停 | 不覆盖系统状态和用户交互 |
| `system` | 自动入睡、唤醒 | 不覆盖用户主动交互 |
| `interaction` | 点击、双击聊天、拖拽、投喂结果 | 可以覆盖其他提示 |

AI 即兴回复会绑定触发它的本地提示序号。如果 AI 回复返回前已经发生新的提示，旧 AI 回复会被丢弃，避免“迟到回复”抢走当前动作或气泡。

## AI 不可用提示

当 AI 即兴调用失败时，桌宠会保留本地台词作为基础反馈，并追加显示“本女王暂时懒得回应。”。

为避免接口异常时频繁刷屏，交互触发的失败提示 8 秒内最多出现一次，闲置等非交互触发的失败提示 30 秒内最多出现一次。

## 拖拽实现

桌宠拖拽由前端手动处理，不使用 `data-tauri-drag-region`。原因是桌宠需要区分单击、双击、拖拽、聊天输入和文件投喂，原生拖拽区域会吞掉一部分点击语义。

当前流程：

1. `pointerdown` 记录 React 指针坐标，用于判断是否超过拖拽阈值。
2. 同时读取 `getCurrentWindow().outerPosition()` 和 `cursorPosition()`，分别作为窗口物理坐标和系统光标物理坐标的起点。
3. `pointermove` 超过 `DRAG_THRESHOLD = 5` 后进入 `drag` 动作。
4. 拖拽中每 `16ms` 轮询一次 `cursorPosition()`，用当前光标物理坐标减去起点光标物理坐标，再叠加起点窗口物理坐标，调用 `setPosition(new PhysicalPosition(...))`。
5. `pointerup` / `pointercancel` 停止轮询，清空拖拽会话，触发 `drop`。

注意事项：

- 不要把 `screenX/screenY` 直接和 `outerPosition()` 混合用于移动；在 Windows 高 DPI 下两者可能处于不同坐标空间，会导致拖拽起步向右下偏移。
- `pet` 窗口必须在 `src-tauri/capabilities/pet.json` 中声明 `core:window:allow-set-position`，否则前端 `setPosition` 构建通过但运行时不会移动。
- `locked = true` 时后端会设置鼠标穿透，桌宠不接收点击和拖拽。

## 代码结构

- `src/components/PetWindow.tsx`：桌宠窗口、动作播放、交互事件、聊天输入、文件投喂入口。
- `src/components/PetWindow.css`：桌宠窗口、气泡、聊天输入、情绪视觉、投喂高亮样式。
- `src/features/petAgent/petCorpus.ts`：本地分场景语料库。
- `src/features/petAgent/petDialogue.ts`：本地语料选择、结构化 AI 回复解析。
- `src/features/petAgent/petPersona.ts`：桌宠人设提示词。
- `src/features/petAgent/petMemory.ts`：运行期短期记忆。
- `src/features/petAgent/petPreferences.ts`：显式长期偏好学习与本地保存。
- `src/features/petAgent/petInput.ts`：搜索框输入预览事件类型与本地点评规则。
- `src/features/ai/chat.ts`：一次性 AI 调用封装 `askOnce`。
- `src/features/pet/petWindow.ts`：桌宠窗口创建、显示、隐藏、位置和尺寸持久化。
- `src-tauri/src/recyclebin.rs`：投喂文件到回收站的后端命令。
- `src-tauri/src/settings.rs`：桌宠设置持久化字段。
- `src-tauri/capabilities/pet.json`：桌宠窗口 Tauri 权限。

## 提示词规则

桌宠提示词会包含：

- 角色身份：她就是《终结的炽天使》中的克鲁鲁·采佩西，以桌宠智能体形态暂居在 Bugzia 桌面上。
- 角色气质：保持克鲁鲁·采佩西的高位吸血鬼女王感，强调粉发、小巧、红瞳、哥特、危险而优雅。
- 核心性格：高傲、冷淡、聪明、腹黑、占有欲强；对长期陪伴的人护短。
- 关系定位：用户是被认可的人类眷属或仆从，不是主人。
- 表达方式：短句优先，像女王下达命令；关心藏在嫌弃或准许里。
- 常用称呼：人类、小家伙、我的眷属、笨蛋人类；若长期偏好里有称呼则优先使用。
- 自称规则：使用“我”或“本女王”，避免过度古风。
- 情绪层次：单击傲娇、双击不满、拖拽抱怨、投喂矜持收下、睡醒有起床气。
- 能力认知：知道自己是桌宠智能体，住在用户桌面上，通过气泡和动作互动。
- 安全边界：不复刻原作台词，不编造未确认的原作剧情，不血腥恐吓，不给出现实伤害指令。
- 当前场景：如点击、聊天、投喂、唤醒。
- 本地语气参考：从语料库抽出的短句。
- 最近互动记忆：运行期最近 3 条互动摘要。
- 长期偏好：用户明确要求保存的称呼或回复风格。
- 用户输入：双击聊天时的输入文本。

AI 输出要求为 JSON：

```json
{
  "line": "中文短句，不超过28个汉字",
  "action": "idle|happy|surprise|wake|annoyed|curious|protective|mocking",
  "mood": "neutral|pleased|annoyed|curious|sleepy|protective|mocking"
}
```

桌宠会根据 `action` 字段切换动作，根据 `mood` 字段更新当前情绪样式类。

## 验收与回归检查

常规构建：

```bash
pnpm build
```

涉及 Tauri capability、窗口权限或后端命令时，还应运行：

```bash
pnpm tauri build
```

手动检查清单：

- 启动桌宠后显示 `wave`，随后回到 `idle`。
- 单击触发 `happy`，双击打开聊天输入框。
- 聊天输入框 Enter 发送，Esc 或 `×` 取消，空内容不触发台词。
- 拖拽时窗口跟随光标，没有起步向右下偏移，拖拽动画持续播放。
- 松开拖拽后触发 `drop`，不会停留在 `drag`。
- 锁定桌宠后鼠标穿透，不能拖拽；解锁后恢复交互。
- 睡眠状态能长期保持，点击后触发 `wake`。
- 搜索框输入关键词能触发对应情绪点评，睡眠、拖拽、锁定时不点评。
- AI 配置缺失或失败时显示“本女王暂时懒得回应。”，不是静默失败。
- 文件拖到桌宠上有投喂悬停反馈，松开后调用回收站投喂逻辑。
